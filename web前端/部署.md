## nginx
Nginx是一个高性能的HTTP和反向代理服务器。
安装：`brew install nginx`
检查配置文件是否有语法错误: `nginx -t`
启动：nginx
停止：nginx -s stop
刷新：nginx -s reload
查看配置：`brew info nginx`
打开安装目录：`open /usr/local/etc/nginx/`
配置文件路径: /usr/local/etc/nginx/nginx.conf
查看Nginx错误日志: `sudo tail -f /var/log/nginx/error.log`
查看访问日志: `sudo tail -f /var/log/nginx/access.log`
开启并查看Nginx状态页面,需要确保Nginx编译时包含了`stub_status`模块,
然后，在Nginx的server配置中添加如下配置：
```
location /nginx_status {
    stub_status;
    allow 127.0.0.1; # 只允许本地访问
    deny all; # 拒绝其他所有来源
}
```
之后重新加载 Nginx: `sudo nginx -s reload`
然后可以通过`http://your_server_ip/nginx_status`访问。

## Docker

## 内网穿透工具Cloudflare

* 安装: `brew install cloudflared`
* 是否安装成功: `cloudflared --version`
* 运行临时穿透 
    `cloudflared tunnel --url http://localhost:3000` 
    得到一个公网URL,URL 每次启动都会变,不适合长期服务。关闭终端即关闭服务
    增加连接`cloudflared tunnel --url http://localhost:3000 --protocol h2mux --connections 4`
* 长期稳定的穿透需登录Cloudflare账号 + **域名**，才能创建穿透生成unnel ID和凭证文件
    * `cloudflared login`打开浏览器登录后有可能不回调
        一直打印：INF Waiting for login...
        如果报错：
        ERR Failed to write the certificate.
        Your browser will download the certificate instead.
        You will have to manually copy it to:
        /Users/xxx/.cloudflared/cert.pem
        error="Failed to fetch resource"
        原因：没有提前在控制面板中添加域名。
    * 验证 cloudflared 是否已登录:`cloudflared tunnel list` 
    * `cloudflared tunnel create myapp`
    * `cloudflared tunnel run myapp`
    * 升级 cloudflared: `brew upgrade cloudflared`
